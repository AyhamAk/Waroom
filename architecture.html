04

Micro-Correction Loop

🔧 Runs within the same cycle · No user wait · Max 3 passes
🏗️
Builder writes files to disk
app.js · data.js · index.html · style.css — validated & written
🎭
testBuild() — Playwright headless
Visits /preview-now · captures console errors + JPEG screenshot · 2.5s settle time
0 errors →  ✅ Break · store screenshot · continue pipeline
errors found →  Categorize: 🔗 DOM · ⚠️ Syntax · 🌐 Network · 💥 Runtime
📋
Build fix prompt
errors list + screenshot + full app.js from disk + element IDs
🤖
Builder fix call · 12 000 tokens
Outputs corrected app.js / data.js · validated before write · whitelist enforced
↑ loop back to testBuild() · max 3 passes total ↑
⏭️
3 passes reached → break
Remaining errors stored in live.consoleErrors for next cycle

Why this matters

Before this loop, console errors only reached the builder in the next cycle — 5+ minutes later. Now the builder self-corrects within seconds of writing its own code, while the full file context is still in its prompt.

Error categories

🔗 DOM — getElementById returning null · wrong element IDs
⚠️ SYNTAX — unexpected token · unclosed braces · parse error
🌐 NETWORK — fetch 404 · CORS · backend unreachable
💥 RUNTIME — TypeError · cannot read property · undefined ref
❓ OTHER — everything else, passed verbatim

Allowed fix targets

public/app.js public/data.js public/index.html public/style.css

Path whitelist enforced — no arbitrary writes during fix passes

05

Automatic Safety Nets

🛡️ Always running · No agent tokens consumed
🔗
CDN Resolver
Replaces hallucinated or wrong-version library URLs in every HTML file with verified cdnjs/jsdelivr URLs from a static map of 35+ libraries.
every HTML write
✅
JS Validator
Runs vm.Script on every JS file before writing. Syntax errors trigger LLM repair via retryBuilderFile().
every JS write
🩹
autoCloseJS
When JS is truncated mid-IIFE (builder hit token limit), tries progressive brace closures until vm.Script passes — no LLM call needed.
before retryBuilderFile()
🔨
TypeScript Stripper
Sucrase transpiler (regex fallback) strips type annotations, interfaces, access modifiers, and generics from LLM-generated JS before validation.
every JS / HTML write
🏥
ensureValidJS
Before each cycle starts, validates app.js and data.js on disk and heals any files corrupted by truncation in the previous cycle.
cycle start (cycle 2+)
📦
ensureRequiredFiles
After every builder run, checks that index.html, style.css, and app.js exist. Auto-generates any missing file with a targeted LLM call.
post-builder, every cycle
🎨
CSS Specialist
After cycle 1, reads the actual HTML, extracts every ID and class, and writes a complete stylesheet targeting exactly those selectors. Runs async.
post cycle 1 (async)
📊
CSS Coverage Validator
Checks what % of HTML selectors have CSS rules. If coverage < 60%, patches style.css targeting only the uncovered selectors.
post-cycle, every cycle (async)
🧠
Agent Memory
Writes docs/agent-memory.json after every cycle: past decisions, built functions, open bugs, backend status. All agents read it next cycle.
post-cycle, every cycle
📋
Feature Inventory
After every builder run, extracts function names and state keys from built JS into docs/features.json so the builder preserves them on rewrite.
post-builder, every cycle
💾
Session Persistence
Saves minimal live state to docs/live-state.json after every cycle. Allows /api/resume-session to recover after a server restart.
post-cycle, every cycle
⏸️
Auto-pause
If all SSE clients disconnect, the pipeline auto-pauses to avoid burning tokens with no observers. Resumes automatically when a client reconnects.
on SSE client disconnect
